# 🔧 Data Loss Recovery Guide - Mess Metric

## What Happened
You accidentally ran `seed.js` which deleted all users from the MongoDB database and only created 3 demo users. This caused:
- ❌ Admin dashboard showing no pending skip requests
- ❌ No student data for skip meals to be requested against
- ❌ Lost student records

## The Root Cause
Your system had 4 main issues:

### 1. **Skip Routes Not Mounted** ❌
The `/api/meals` endpoint was not registered in `server/index.js`, so:
- Admin dashboard couldn't fetch pending skip requests
- Students' skip requests had no endpoint to submit to

### 2. **Missing GET Endpoint** ❌
The skip controller only had a `requestSkip` (POST) function but no `getPendingSkipRequests` (GET) function for admins

### 3. **Frontend Not Calling Skip API** ❌
The `DailyMealTracker.jsx` had the skip request call commented out

### 4. **Missing walletAddress Field** ❌
Admin dashboard needed `walletAddress` from users for minting coins, but the User model was missing this field

## 📋 API Endpoints Summary

### Student Endpoints
```
POST /api/meals/skip
  - Headers: Authorization: Bearer {token}
  - Body: { mealType: "Breakfast|Lunch|Snacks|Dinner" }
  - Response: { success: true, requestId: "..." }
```

### Admin Endpoints
```
GET /api/meals
  - Headers: Authorization: Bearer {token}
  - Checks: req.user.role === "admin"
  - Response: { success: true, data: [{ id, student, mealType, status, ... }] }

PUT /api/meals/{id}/approve
  - Headers: Authorization: Bearer {token}
  - Checks: req.user.role === "admin"
  - Response: { success: true, message: "Skip request approved", data: {...} }
```

## 🐛 Troubleshooting

### Admin Dashboard Still Empty?
```
1. Check browser Network tab (F12 > Network)
2. Click GET request to /api/meals
3. Check Response tab - should show pending requests
4. If error 403: Admin JWT missing role: "admin"
5. If error 401: No token provided or expired token
```

### Skip Requests Not Submitting?
```
1. Check console (F12 > Console)
2. Look for Skip request submitted message
3. Check Network tab for POST to /api/meals/skip
4. Verify response status is 200
```

### Database Empty After seed.js?
```
1. Ensure MongoDB is running
2. Check .env file has correct MONGO_URI
3. Run seed.js again: node seed.js
4. Can optionally run: npm run seed (if script exists in package.json)
```
